脚本说明：
根据z510 Story2 case表格设计的
环境要求： 2台小站，近端和远端
         近端 3个摄像头本地管理配置教师和学生算法,板书提取算法

         远端 HDMI 1个摄像头

********************************************************************************
重要说明：

推流方式：(比如)python remote_push.py -ct=HDMI -ip=10.12.224.44 -cn=1
        -d指定对应小站推流的摄像头，-ip为对应小站id，-ct 为推流摄像头类型 -cn 为摄像头序列
        可以用python delete_all_tasks.py -ip=10.12.224.44删除对应小站的任务
********************************************************************************

运行方式与执行步骤
Case 01：
1.终端输入python 11.2.1_4F.py
2.终端输入python student_push.py -ip={远端小站IP} ，让远端小站进行推流
2.创建四宫回显，1导播2学生3教师4远端拉流，同时启动对导播路音视频录制。
3.删除任务(python delete_all_task.py -ip={对应小站IP})后检测录制的视频文件 音视频效果是否正常

Case 02：
1.在上一个Case的基础上，执行上一个步骤1 进行测试
2.终端输入python 11.2.2_change_position.py 变化宫格位置，1-4互换，2-3互换
3.删除任务，检查录制的文件 音视频是否正常

Case 03：
1.在Case01 的基础上，执行Case 01步骤1 进行测试
2.终端输入python 11.2.3_change_source.py  更改回显视频源，1宫为导播，2宫教师AI，3宫板书AI，四宫学生AI
3.删除任务，检查录制的文件 音视频是否正常

Case 04：
1.在Case01 的基础上，执行Case 01步骤1 进行测试
2.继续执行Case01 的步骤2
